# Relay — App Store Optimization

Store listing copy + ASO strategy for the App Store and Google Play. Brand name
stays clean ("Relay"); high-intent keywords live in the subtitle, description,
and keyword field.

## Names & subtitles

**App name (iOS, ≤30 chars):** `Relay: Auto Text & AI Reply`
**Subtitle (iOS, ≤30 chars):** `Business texting on autopilot`

**Title (Google Play, ≤30 chars):** `Relay – Auto Text & Replies`
**Short description (Play, ≤80 chars):**
`Schedule texts to customer groups and let AI reply 24/7. Built for small biz.`

## Keyword field (iOS, ≤100 chars, comma-separated, no spaces/dupes)
```
sms,text,autoreply,auto,reply,schedule,marketing,business,customer,reminder,whatsapp,crm,bulk,campaign
```
> Don't repeat words already in the name/subtitle (Apple indexes those
> separately). Singular forms; Apple handles plurals.

> Download Relay and never miss a customer again.

Use real-device frames, a short benefit caption on each, brand gradient
background, first 2 shots must sell it (most users never swipe further).

## Metadata
- **Category:** Business (primary), Productivity (secondary)
- **Age rating:** 4+
- **Privacy:** Contacts (optional, for importing), customer phone numbers &
  message content (stored to operate the service). Publish a privacy policy and
  a data-safety form; declare no third-party ad tracking.

## ASO best-practices checklist
- [x] Keyword-rich subtitle/short description, brand stays clean
- [x] Primary keywords in the first sentence of the long description
- [x] No keyword stuffing or competitor names
- [x] Distinct, benefit-led screenshot captions; first two carry the pitch
- [x] Promotional text used for updates (no resubmission)
- [ ] Localize name/keywords/screenshots for top markets
- [ ] Iterate keywords from Search Ads / Play Console search-term data
